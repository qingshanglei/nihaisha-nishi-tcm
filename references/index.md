# Nihaixia 中医资料索引

本 skill 用于按课程模块渐进加载资料，避免一次性加载全部转写。当前已完成 `07.伤寒论课程` 的总览、细分模块、逐课地图和截图证据索引。

## 伤寒论模块

| 文件 | 用途 | 何时打开 |
| --- | --- | --- |
| `learning-entry.md` | 用户学习首页，按“学这个为了做什么”组织 | 判断用户目的、设计用户入口、评估结构是否适合学习时 |
| `beginner-questions.md` | 普通用户白话问法入口 | 用户不懂中医术语，用“感冒、拉肚子、手脚冷、睡不着”等日常语言提问时 |
| `usage-scenarios.md` | 用户使用场景路由 | 不确定用户入口，或要设计回答结构时 |
| `six-channel.md` | 六经辨证细模块 | 问太阳/阳明/少阳/三阴病，或症状需先定位层次 |
| `formula-patterns.md` | 方剂方证细模块 | 问某个方、方证鉴别、禁忌、相关方 |
| `symptom-index.md` | 症状入口索引 | 用户从发热、便秘、下利、腹痛、失眠、手足冷等症状提问 |
| `lesson-map.md` | 逐课学习地图 | 问第几讲、学习计划、从某课开始复习 |
| `screenshot-evidence.md` | 649 张关键截图索引 | 用户要板书/PPT截图佐证，或需要定位视频笔记证据 |
| `shanghanlun.md` | 课程总纲旧版整合文档 | 需要一次性总览课程结构、核心观点和原始总表 |

## 检索建议

- 症状问题：先 `symptom-index.md`，再 `six-channel.md`，最后 `formula-patterns.md`。
- 白话问题：先 `beginner-questions.md`，把普通话问题转成分水岭，再进入症状/六经/方证模块。
- 方剂问题：先 `formula-patterns.md`，必要时回到 `six-channel.md` 看病机层次。
- 逐课问题：用 `lesson-map.md`，再按课次查 `screenshot-evidence.md`。
- 截图问题：用 `rg "关键词" references/screenshot-evidence.md`，关键词可以是方名、病机、六经名、课次或时间点。
- 结构设计问题：先看 `learning-entry.md`，优先按用户任务组织，而不是按课程目录组织。
- 医疗安全：真实症状只做课程学习分析，不直接给个人处方；高风险方药和急危重症必须提醒面诊或急诊。
